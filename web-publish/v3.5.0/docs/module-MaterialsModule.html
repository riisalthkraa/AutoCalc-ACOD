MaterialsModule

======================================== MATERIALS MODULE - Gestion des matériaux

Module de gestion complète du catalogue de matériaux pour l'application AutoCalc OptiDevis. Gère tous les aspects liés aux matériaux : chargement, affichage, recherche, filtrage, CRUD (Create, Read, Update, Delete), et sélection pour devis.

Architecture : Le module fonctionne sur deux onglets distincts avec des rôles différents :

  1. Onglet Recherche (tous les utilisateurs) :

    • Recherche rapide de matériaux par nom, référence, type
    • Filtrage par catégorie avec compteur
    • Sélection de matériaux pour les ajouter au devis en cours
    • Interface simplifiée optimisée pour la recherche

  2. Onglet Catalogue (mode Admin uniquement) :

    • Tableau complet des matériaux avec toutes les colonnes
    • Ajout, édition, suppression de matériaux
    • Filtrage avancé (recherche texte + type + fournisseur)
    • Gestion complète du catalogue

Workflow principal - Ajout d'un matériau :

  1. Utilisateur clique sur "Ajouter un matériau" (onglet Catalogue)
  2. Modal s'ouvre avec formulaire vide
  3. Utilisateur remplit les champs (nom, type, prix TTC, TVA, etc.)
  4. Calcul automatique HT ← TTC et TVA
  5. Utilisateur clique "Enregistrer"
  6. Validation des champs obligatoires (nom, type)
  7. Appel IPC 'addMaterial' avec sessionToken
  8. Invalidation du cache + rechargement
  9. Rafraîchissement de la table + notification succès

Workflow principal - Édition d'un matériau :

  1. Utilisateur clique sur bouton "Modifier" d'un matériau
  2. Modal s'ouvre avec formulaire prérempli
  3. Utilisateur modifie les champs souhaités
  4. Utilisateur clique "Enregistrer"
  5. Appel IPC 'updateMaterial' avec sessionToken
  6. Invalidation du cache + rechargement
  7. Rafraîchissement de la table + notification succès

Workflow principal - Suppression d'un matériau :

  1. Utilisateur clique sur bouton "Supprimer" d'un matériau
  2. Modal de confirmation s'affiche
  3. Utilisateur confirme la suppression
  4. Appel IPC 'deleteMaterial' avec sessionToken
  5. Invalidation du cache + rechargement
  6. Rafraîchissement de la table + notification succès

Workflow principal - Sélection pour devis :

  1. Utilisateur recherche un matériau dans l'onglet Recherche
  2. Résultats affichés en temps réel avec filtrage
  3. Utilisateur clique sur "Sélectionner" sur un matériau
  4. Matériau stocké dans state.selectedMaterial + window.selectedMaterial
  5. Affichage mis à jour dans le tableau de bord Vendeur
  6. QuoteBuilderModule peut utiliser ce matériau pour calculer le devis

Système de cache :

  • Cache DataCache de 5 minutes pour les matériaux
  • Invalidation automatique lors de tout changement (ajout/modif/suppression)
  • Protection contre l'écrasement : si backend retourne [], on garde les matériaux existants

Système de filtrage :

  • Recherche texte : nom, référence, type
  • Filtre par type : dropdown avec compteur par catégorie
  • Filtre par fournisseur : activé depuis SuppliersModule (filterBySupplier)
  • Filtres cumulatifs (ET logique)

Calcul automatique des prix :

  • L'utilisateur saisit le prix TTC et la TVA
  • Le prix HT est calculé automatiquement : HT = TTC / (1 + TVA/100)
  • Lors de l'affichage : TTC = HT * (1 + TVA/100)

Compatibilité avec legacy code :

  • Exposé via window.materials pour quick-add.js et autre ancien code
  • Synchronisation bidirectionnelle : window.materials ↔ this.materials ↔ state.materials
  • Support de window.selectedMaterial pour compatibilité
  • Fonctions exposées au scope global via exposeToGlobal()

Événements émis :

  • 'materials:initialized' - Module initialisé
  • 'materials:loaded' - Matériaux chargés depuis backend/cache
  • 'materials:added' - Matériau ajouté
  • 'materials:updated' - Matériau modifié
  • 'materials:deleted' - Matériau supprimé
  • 'materials:selected' - Matériau sélectionné pour devis

Événements écoutés :

  • 'suppliers:loaded' - Rafraîchir la liste des fournisseurs
  • 'suppliers:updated' - Rafraîchir la liste des fournisseurs

Canaux IPC utilisés :

  • 'get-materials' - Récupérer tous les matériaux
  • 'addMaterial' - Ajouter un nouveau matériau (nécessite sessionToken)
  • 'updateMaterial' - Modifier un matériau existant (nécessite sessionToken)
  • 'deleteMaterial' - Supprimer un matériau (nécessite sessionToken)

Intégrations :

  • SuppliersModule : Liste des fournisseurs pour le select + filtrage
  • QuoteBuilderModule : Matériau sélectionné pour calcul de devis
  • QuickAddModule : Ajout rapide de matériaux (via window.materials)
  • ScannerModule : Recherche par code-barres
  • Dashboard Vendeur : Affichage du matériau sélectionné
