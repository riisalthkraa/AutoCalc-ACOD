# async deleteClient(id) → {Promise.<void>}

Supprime définitivement un client après confirmation. Envoie la requête de suppression via IPC, invalide le cache et rafraîchit.

Workflow :

  1. Validation : ID non vide (sinon error + return)
  2. Récupère sessionToken pour authentification IPC
  3. Appelle IPC 'deleteClient' avec sessionToken + id
  4. Si succès :
    • Affiche NotificationSystem.success
    • Émet événement 'deleted' avec id
    • Ferme le modal de confirmation (#deleteClientModal)
    • Invalide cache DataCache ('clients')
    • Recharge les clients (loadClients)
    • Rafraîchit le tableau (populateClientsTable)
    • Émet événement global 'clients:updated' (EventBus)
    • Réinitialise this.clientToDeleteId
  5. Si erreur → throw Error
Parameters:
Name Type Description
id number | string

ID du client à supprimer

# async saveClient() → {Promise.<void>}

Sauvegarde un client (ajout ou modification selon editMode). Valide le formulaire, envoie via IPC, invalide le cache et rafraîchit.

Workflow :

  1. Récupère les 10 champs du formulaire (via getValue())
  2. Validation : nom obligatoire (sinon error + return)
  3. Récupère sessionToken pour authentification IPC
  4. Si editMode = true :
    • Récupère l'ID du client (#client-id)
    • Appelle IPC 'updateClient' avec sessionToken + clientData
    • Émet événement 'updated' avec clientData
  5. Si editMode = false :
    • Appelle IPC 'addClient' avec sessionToken + clientData
    • Émet événement 'added' avec clientData
  6. Invalide le cache DataCache ('clients')
  7. Recharge les clients (loadClients)
  8. Rafraîchit le tableau (populateClientsTable)
  9. Émet événement global 'clients:updated' (EventBus)
  10. Nettoie le formulaire (clearClientForm)

View Source modules/clients/ClientsModule.js, line 1072

  • NotificationSystem#event:error Si nom manquant ou erreur IPC
  • NotificationSystem#event:success Si sauvegarde réussie
  • EventBus#added Après ajout réussi (avec clientData)
  • EventBus#updated Après modification réussie (avec clientData)
  • clients:updated Événement global pour notifier autres modules
Promise.<void>

# viewClientQuotes(clientName) → {void}

Bascule vers l'onglet Catalogue et filtre les devis et factures par client. Utilise Bootstrap Tab API pour changer d'onglet et appelle filterByClientName() du MaterialsModule.

Workflow :

  1. Récupère l'onglet #admin-tab (Catalogue)
  2. Active l'onglet via Bootstrap Tab API (new bootstrap.Tab().show())
  3. Attend 100ms pour laisser l'onglet se charger
  4. Appelle window.filterByClientName(clientName) du MaterialsModule
  5. Si fonction non disponible → affiche warning dans console

Intégration :

  • Appelé par le bouton "Voir devis et factures" dans le tableau des clients
  • Affiche le badge vert avec nombre de devis et factures (ex: "12")
  • Nécessite que MaterialsModule expose filterByClientName() en global
