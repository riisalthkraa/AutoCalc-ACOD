# constant crypto

CONTEXTE DE SÉCURITÉ : ENCRYPTION AT REST

Les mots de passe SMTP ne doivent JAMAIS être stockés en clair dans config.json. Ce module utilise AES-256-CBC pour chiffrer/déchiffrer les mots de passe.

PROTECTIONS IMPLEMENTÉES :

  1. Chiffrement AES-256-CBC (algorithme standard NSA)
  2. Clé dérivée avec scrypt (KDF sécurisé)
  3. IV aléatoire pour chaque chiffrement (empêche analyse)
  4. Gestion sécurisée de la clé de chiffrement

IMPORTANT :

  • La clé de chiffrement doit être stockée dans une variable d'environnement
  • Ne JAMAIS commiter la clé dans Git
  • Changer la clé si elle est compromise
