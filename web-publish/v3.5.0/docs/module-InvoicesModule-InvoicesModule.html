# async askCreateInvoice

Demande la création d'une facture depuis un devis accepté.

ADMIN UNIQUEMENT - Nécessite l'authentification Admin.

Workflow :

  1. Vérifier l'authentification Admin (isAuthenticated)
  2. Si non authentifié : notification + arrêt
  3. Charger le devis via IPC 'load-saved-quote'
  4. Stocker le devis dans this.currentQuoteForInvoice
  5. Vérifications :
    • Statut doit être "accepté" (sinon erreur)
    • Pas déjà facturé (vérifier invoiceId via hasValidInvoiceId)
  6. Remplir le modal de confirmation :
    • Nom du devis
    • Nom du client
    • Total TTC (formaté en devise)
  7. Afficher le modal Bootstrap

L'utilisateur peut ensuite confirmer (→ confirmCreateInvoice) ou annuler.

# async confirmCreateInvoice

Confirme la création de la facture depuis un devis accepté.

Workflow :

  1. Récupérer authToken (getSessionToken)
  2. Vérifier qu'il existe (sinon erreur)
  3. Vérifier que this.currentQuoteForInvoice est défini
  4. Appeler IPC 'create-invoice-from-quote' avec authToken + quoteId
  5. Si succès :
    • Notification de succès avec numéro de facture
    • Fermer le modal
    • Émettre événement 'invoices:created'
    • Attendre 300ms
    • Rafraîchir liste des devis (refreshSavedQuotesTable)
    • Rafraîchir onglet Devis si actif (loadQuotesInTab)
    • Rafraîchir liste des factures si actif (loadInvoices)
  6. Réinitialiser this.currentQuoteForInvoice

Note : Le backend crée automatiquement la facture avec :

  • Numéro unique (FAC-YYYYMMDD-XXXX)
  • Date émission = aujourd'hui
  • Date échéance = émission + 30 jours
  • Statut = à_payer
  • Mise à jour du devis : statut → facturé, invoiceId stocké

# async sendInvoiceEmail

Envoie une facture par email au client.

ADMIN UNIQUEMENT - Nécessite authToken.

Workflow :

  1. Récupérer authToken (getSessionToken)
  2. Si non authentifié : notification erreur + arrêt
  3. Charger la facture via IPC 'load-invoice'
  4. Si erreur : notification + arrêt
  5. Vérifier que l'email client est renseigné
  6. Si non renseigné : notification erreur + arrêt
  7. Générer le PDF via window.generateInvoicePDFDocument(invoice)
  8. Si fonction non disponible : notification erreur + arrêt
  9. Convertir le PDF en Blob
  10. Lire le Blob en base64 via FileReader
  11. Préparer les données email :
    • to: email du client
    • subject: "Facture {numéro}"
    • body: message standard
    • attachmentName: facture_{numéro}.pdf
    • attachmentData: base64
  12. Appeler IPC 'send-email' avec authToken + emailData
  13. Si succès : notification + événement 'invoices:emailed'
  14. Si échec : notification erreur

# hasValidInvoiceId(invoiceId) → {boolean}

Vérifie si un invoiceId est valide.

Un invoiceId est considéré valide s'il n'est pas :

  • null
  • undefined

# renderInvoiceRow(invoice) → {string}

Génère le HTML d'une ligne de facture pour le tableau.

Colonnes affichées :

  • Numéro de facture (gras)
