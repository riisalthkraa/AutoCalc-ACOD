SuppliersModule

======================================== SUPPLIERS MODULE - Gestion des fournisseurs

Module de gestion complète des fournisseurs pour l'application AutoCalc OptiDevis. Gère le catalogue de fournisseurs avec opérations CRUD, filtrage avancé et liaison avec les matériaux du catalogue.

Architecture : Le module gère un catalogue de fournisseurs avec :

  • CRUD complet : Create, Read, Update, Delete avec validation
  • Filtrage multi-critères : Recherche textuelle + filtre par type
  • Liaison matériaux : Affichage du nombre de matériaux par fournisseur
  • Cache intelligent : DataCache 5 minutes pour performance
  • Modals Bootstrap : Formulaire d'ajout/édition + confirmation suppression

========================================== DONNÉES FOURNISSEUR

Structure d'un fournisseur (11 champs) :

  • id : Identifiant unique (auto-incrémenté)
  • name : Nom du fournisseur (obligatoire, unique)
  • contact : Nom du contact principal
  • email : Adresse email du fournisseur
  • phone : Numéro de téléphone
  • address : Adresse complète (tronquée à 50 caractères dans tableau)
  • type : Type de fournisseur (Maçonnerie, Plomberie, Électricité, etc.)
  • siret : Numéro SIRET (identifiant légal)
  • deliveryDelay : Délai de livraison en jours (nombre)
  • website : Site web du fournisseur
  • notes : Notes libres (tronquées à 30 caractères dans tableau)

Stockage : suppliers.json via IPC

========================================== FONCTIONNALITÉS CRUD

Create (Ajout) :

  1. Utilisateur clique "Ajouter fournisseur"
  2. Modal s'ouvre avec formulaire vide
  3. Utilisateur remplit les champs (nom obligatoire)
  4. Validation : nom non vide
  5. Envoi via IPC 'addSupplier' avec authToken
  6. Invalidation cache + rechargement + rafraîchissement tableau
  7. Notification succès

Read (Lecture) :

  • Chargement au démarrage depuis cache ou backend (IPC 'get-suppliers')
  • Affichage dans tableau trié par ID
  • Comptage des matériaux par fournisseur (liaison avec catalog)
  • Bouton "Voir matériaux" pour basculer vers onglet Catalogue avec filtre actif

Update (Modification) :

  1. Utilisateur clique icône "Modifier" sur une ligne
  2. Modal s'ouvre avec formulaire pré-rempli
  3. Utilisateur modifie les champs
  4. Validation : nom non vide
  5. Envoi via IPC 'updateSupplier' avec authToken + supplierData.id
  6. Invalidation cache + rechargement + rafraîchissement
  7. Notification succès

Delete (Suppression) :

  1. Utilisateur clique icône "Supprimer"
  2. Modal de confirmation avec nom du fournisseur
  3. Confirmation utilisateur
  4. Envoi via IPC 'deleteSupplier' avec authToken + id
  5. Invalidation cache + rechargement + rafraîchissement
  6. Fermeture modal + notification succès

========================================== FILTRAGE & RECHERCHE

Recherche textuelle (#search-suppliers) :

  • Recherche dans : name, contact, email, phone
  • Insensible à la casse
  • Temps réel (événement 'input')

Filtre par type (#filter-supplier-type) :

  • Select généré dynamiquement avec comptage par type
  • Option "Tous les types" par défaut
  • Format : "Type (nombre)"
  • Événement 'change'

Combinaison des filtres :

  • Les deux filtres se cumulent (ET logique)
  • Notification du nombre de résultats (toast 2s)
  • Affichage "Aucun fournisseur trouvé" si 0 résultat

========================================== LIAISON AVEC MATÉRIAUX

Intégration avec MaterialsModule :

  • Comptage des matériaux par fournisseur (material.supplier = supplier.name)
  • Badge vert avec nombre si matériaux > 0
  • Bouton "Voir matériaux" avec icône œil
  • Fonction viewSupplierMaterials() :
    1. Bascule vers onglet Catalogue (Bootstrap Tab API)
    2. Appelle filterBySupplier() du MaterialsModule avec délai 100ms
    3. Filtre automatiquement le catalogue par fournisseur

Utilité : Permet de voir rapidement quels matériaux viennent d'un fournisseur

========================================== CACHE & PERFORMANCE

DataCache avec TTL 5 minutes :

  • Clé : 'suppliers'
  • Durée : 5 * 60 * 1000 ms = 300 secondes
  • Vérification au loadSuppliers() :
    • Si cache valide → utilise cache
    • Si cache expiré → fetch backend + mise en cache
  • Invalidation manuelle après CRUD :
    • DataCache.invalidate('suppliers')
    • Force rechargement depuis backend

Synchronisation :

  • this.suppliers (instance)
  • this.state.set('suppliers') (StateManager)
  • window.suppliers (global, compatibilité legacy)
  • DataCache ('suppliers')

========================================== MODALS BOOTSTRAP

Modal Formulaire (#supplierFormModal) :

  • Titre dynamique : "Ajouter" ou "Modifier"
  • 11 champs de saisie
  • Boutons : Sauvegarder, Annuler
  • Focus automatique sur #supplier-name après ouverture (délai 500ms)
  • Mode : this.editMode (boolean)

Modal Suppression (#deleteSupplierModal) :

  • Message dynamique avec nom du fournisseur
  • Boutons : Confirmer, Annuler
  • ID stocké temporairement dans this.supplierToDeleteId

========================================== EVENT LISTENERS

Listeners globaux (attachEventListeners) :

  • Onglet #suppliers-tab → shown.bs.tab : Rafraîchit tableau + attache listeners

Listeners spécifiques (attachSuppliersListeners) :

  • Protection contre double-attachement (suppliersListenersAttached flag)
  • Boutons : #add-supplier-btn, #save-supplier-btn, #cancel-supplier-btn, #confirmDeleteSupplierBtn
  • Filtres : #search-suppliers (input), #filter-supplier-type (change)
  • Tableau : onclick editSupplier(id), askDeleteSupplier(id), viewSupplierMaterials(name)

========================================== ÉVÉNEMENTS ÉMIS

EventBus interne :

  • 'initialized' : Au démarrage du module
  • 'loaded' : Après chargement des fournisseurs (avec data)
  • 'added' : Après ajout réussi (avec supplierData)
  • 'updated' : Après modification réussie (avec supplierData)
  • 'deleted' : Après suppression réussie (avec id)

EventBus global (this.events) :

  • 'suppliers:loaded' : Pour notifier autres modules du chargement
  • 'suppliers:updated' : Après tout changement (add/update/delete)

Utilité : MaterialsModule peut écouter 'suppliers:loaded' pour rafraîchir son autocomplete

========================================== INTÉGRATIONS

  • MaterialsModule : Liaison par supplier name, fonction filterBySupplier()
  • StateManager : Stockage suppliers, editMode, supplierToDeleteId
  • DataCache : Cache 5 min pour performance
  • NotificationSystem : Feedback utilisateur (success, error, info)
  • Bootstrap 5.3.3 : Modals, Tab API, Tooltips
  • IPC Electron : get-suppliers, addSupplier, updateSupplier, deleteSupplier
