# analyzeProductData

Analyse les données des produits depuis les devis et factures pour générer des statistiques

ALGORITHME:

  1. Parcourt les devis acceptés/facturés ET factures payées
  2. Agrège par produit (nom) et catégorie (type)
  3. Calcule quantités vendues + CA par produit
  4. Compare mois actuel vs mois précédent pour identifier tendances
  5. Génère données pour graphiques "Top Produits" et "Répartition Catégories"

# displayTrendingProducts

Affiche les produits en tendance (hausse/baisse) en comparant mois actuel vs mois précédent

LOGIQUE DE DÉTECTION DES TENDANCES:

  • Hausse: Variation > +5% OU nouveau produit avec CA > 0
  • Baisse: Variation < -5% OU produit disparu (0 ventes)
  • Calcul: ((CA actuel - CA précédent) / CA précédent) × 100

SEUILS:

  • Variation significative: ±5%
  • Affichage limité au Top 5 hausse + Top 5 baisse

# generateRecommendation

Génère des recommandations intelligentes basées sur les tendances produits.

Analyse le type de tendance (hausse/baisse) et l'ampleur de la variation pour suggérer des actions concrètes aux administrateurs.

Recommandations en hausse :

  • Nouveau produit → Mise en avant page d'accueil
  • Variation > 50% → Vérifier stocks + promotion croisée
  • Variation > 20% → Mise en avant auprès des vendeurs
  • Variation > 5% → Continuer le suivi

Recommandations en baisse :

  • Produit disparu → URGENT : Enquêter auprès des vendeurs
  • Variation > 50% → Action requise : prix/qualité/concurrence
  • Variation > 30% → Interviewer vendeurs pour comprendre freins
  • Variation > 5% → Surveiller l'évolution

View Source modules/dashboard/DashboardModule.js, line 1670

Example
const recommendation = this.generateRecommendation(
  { name: "Parpaing", current: 5000, previous: 3000, change: 66.7 },
  'up'
);
// → "Forte demande ! Vérifier les stocks et envisager une promotion croisée."

# async updateAdminKPIs

Met à jour les 4 KPIs du dashboard administrateur.

KPIs calculés :

  1. CA du mois : Somme des totaux TTC des factures payées du mois en cours
  2. Devis en attente : Nombre de devis avec status "en_attente" (avec badge rouge)
  3. Factures en retard : Factures avec status "échue" OU dueDate dépassée (avec badge rouge)
  4. Taux de conversion : % de devis acceptés ou facturés / total devis

Éléments DOM mis à jour :

  • #kpi-ca-month : CA du mois (€)
  • #kpi-quotes-pending : Nombre de devis en attente
  • #admin-pending-badge : Badge rouge (visible si > 0)
  • #kpi-invoices-overdue : Nombre de factures en retard
  • #admin-overdue-badge : Badge rouge (visible si > 0)
  • #kpi-conversion-rate : Taux de conversion (%)

# updateVendeurTopMaterials

Met à jour le tableau "Top 10 Articles" du dashboard vendeur.

Algorithme :

  1. Filtre l'historique pour garder les recherches des 7 derniers jours
  2. Compte les occurrences par matériau (materialId)
  3. Trie par nombre de recherches (décroissant)
  4. Prend le top 10
  5. Génère le HTML du tableau avec boutons "Sélectionner"

Affiche "Aucune donnée disponible" si aucune recherche cette semaine.

Colonnes du tableau :

  • Rang (1-10)
  • Nom du matériau
  • Catégorie (badge gris)
  • Nombre de recherches (badge bleu)
  • Prix unitaire
  • Bouton "Sélectionner" (appelle selectMaterial(id))
