# findMaterial

Recherche un matériau avec algorithme fuzzy multi-niveaux. Utilise 6 niveaux de recherche progressive du plus strict au plus permissif.

Stratégie de recherche (par ordre de priorité) :

  1. Exacte : name === term OU reference === term
  2. Exacte nettoyée : Après suppression stop words
  3. Partielle : name.includes(term) OU reference.includes(term) OU type.includes(term)
  4. Partielle nettoyée : Après suppression stop words
  5. Fuzzy : TOUS les mots-clés présents (every)
  6. Permissive : AU MOINS 1 mot-clé présent (some)

Stop words ignorés :

  • Articles : de, le, la, les, un, une, des, du
  • Contenants : sac, sacs, plaque, plaques, barre, barres, rouleau, rouleaux

// Recherche exacte
findMaterial("parpaing");
// → Trouve "Parpaing" (niveau 1)
// Recherche avec stop words

// → Trouve "Plâtre" (niveau 3)

# processInput

Traite l'entrée utilisateur du Quick Add et ajoute le matériau au devis. Parse la syntaxe naturelle, recherche le matériau, calcule la quantité et ajoute au devis.

Syntaxe : [quantité][unité optionnelle] [nom matériau]

Workflow :

  1. Normalisation input (m2→m², m3→m³, virgule→point)
  2. Parse avec regex : /^(\d+(?:[.,]\d+)?)\s*(?:(m²|m³|m|/h))?\s+(.+)$/i
  3. Si format invalide → NotificationSystem.warning + return
  4. Recherche matériau via findMaterial() (fuzzy multi-niveaux)
  5. Si introuvable → error + suggestMaterials() + return
  6. Calcul quantité via calculateQuantity() (conversions unités)
  7. Ajout au devis via addToQuote()
  8. Vide input et refocus
