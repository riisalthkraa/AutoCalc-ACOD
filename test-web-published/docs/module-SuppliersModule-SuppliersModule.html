# applySupplierFilters() → {void}

Applique les filtres de recherche et de type sur les fournisseurs. Combine recherche textuelle ET filtre par type (logique AND). Affiche le nombre de résultats via notification.

# async deleteSupplier(id) → {Promise.<void>}

Supprime définitivement un fournisseur après confirmation. Envoie la requête de suppression via IPC, invalide le cache et rafraîchit.

Workflow :

  1. Validation : ID non vide (sinon error + return)
  2. Récupère sessionToken pour authentification IPC
  3. Appelle IPC 'deleteSupplier' avec sessionToken + id
  4. Si succès :
    • Affiche NotificationSystem.success
    • Émet événement 'deleted' avec id
    • Ferme le modal de confirmation (#deleteSupplierModal)
    • Invalide cache DataCache ('suppliers')
    • Recharge les fournisseurs (loadSuppliers)
    • Rafraîchit le tableau (populateSuppliersTable)
    • Émet événement global 'suppliers:updated' (EventBus)
    • Réinitialise this.supplierToDeleteId
  5. Si erreur → throw Error

# async saveSupplier() → {Promise.<void>}

Sauvegarde un fournisseur (ajout ou modification selon editMode). Valide le formulaire, envoie via IPC, invalide le cache et rafraîchit.

Workflow :

  1. Récupère les 10 champs du formulaire (via getValue())
  2. Validation : nom obligatoire (sinon error + return)
  3. Récupère sessionToken pour authentification IPC
  4. Si editMode = true :
    • Récupère l'ID du fournisseur (#supplier-id)
    • Appelle IPC 'updateSupplier' avec sessionToken + supplierData
    • Émet événement 'updated' avec supplierData
  5. Si editMode = false :
    • Appelle IPC 'addSupplier' avec sessionToken + supplierData
    • Émet événement 'added' avec supplierData
  6. Invalide le cache DataCache ('suppliers')
  7. Recharge les fournisseurs (loadSuppliers)
  8. Rafraîchit le tableau (populateSuppliersTable)
  9. Émet événement global 'suppliers:updated' (EventBus)
  10. Nettoie le formulaire (clearSupplierForm)

View Source modules/suppliers/SuppliersModule.js, line 995

  • NotificationSystem#event:error Si nom manquant ou erreur IPC
  • NotificationSystem#event:success Si sauvegarde réussie
  • EventBus#added Après ajout réussi (avec supplierData)
  • EventBus#updated Après modification réussie (avec supplierData)
  • suppliers:updated Événement global pour notifier autres modules
Promise.<void>

# viewSupplierMaterials(supplierName) → {void}

Bascule vers l'onglet Catalogue et filtre les matériaux par fournisseur. Utilise Bootstrap Tab API pour changer d'onglet et appelle filterBySupplier() du MaterialsModule.

Workflow :

  1. Récupère l'onglet #admin-tab (Catalogue)
  2. Active l'onglet via Bootstrap Tab API (new bootstrap.Tab().show())
  3. Attend 100ms pour laisser l'onglet se charger
  4. Appelle window.filterBySupplier(supplierName) du MaterialsModule
  5. Si fonction non disponible → affiche warning dans console

Intégration :

  • Appelé par le bouton "Voir matériaux" dans le tableau des fournisseurs
  • Affiche le badge vert avec nombre de matériaux (ex: "12")
  • Nécessite que MaterialsModule expose filterBySupplier() en global
