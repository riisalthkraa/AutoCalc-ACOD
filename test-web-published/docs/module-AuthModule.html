AuthModule

======================================== AUTH MODULE - Gestion de l'authentification

Module de gestion de l'authentification et des permissions utilisateur. Gère le switch entre mode Administrateur (accès complet) et mode Vendeur/Conseil (lecture seule, fonctionnalités limitées).

Architecture :

  • Authentification par mot de passe (admin uniquement)
  • Gestion du token de session sécurisé
  • Configuration dynamique de l'interface selon le mode
  • Démarrage sécurisé en mode Vendeur/Conseil
  • Pas de persistance du mode admin (sécurité)

Responsabilités :

  • Gérer les sessions utilisateur (login/logout)
  • Vérifier le mot de passe via IPC sécurisé
  • Générer et stocker le token de session
  • Basculer entre mode Admin et mode Vendeur/Conseil
  • Configurer l'interface selon les permissions (afficher/masquer onglets)
  • Synchroniser l'état d'authentification avec StateManager
  • Exposer les variables globales pour compatibilité legacy

Modes d'utilisation :

Mode Vendeur/Conseil (par défaut au démarrage) :

  • Accès à : Recherche, Catalogue, Construction devis, Accueil (dashboard vendeur)
  • Masqué : Admin, Factures, Configuration, Templates, Fournisseurs, CA du Mois, Tableau de bord
  • Permissions : Lecture seule, pas d'export PDF, pas d'envoi email
  • Sécurité : Aucune authentification requise

Mode Administrateur (après authentification) :

  • Accès à : Tous les onglets
  • Permissions : Lecture + Écriture, Export PDF, Envoi email, Gestion complète
  • Sécurité : Mot de passe requis, token de session généré
  • Session : Non persistée, l'utilisateur doit se réauthentifier à chaque démarrage

Workflow d'authentification :

1. Démarrage app → Mode Vendeur/Conseil automatique
2. Utilisateur active le switch "Mode Admin"
3. Modal de mot de passe s'affiche
4. Utilisateur saisit le mot de passe
5. Vérification IPC 'verify-password'
6. Si valide → génération token + passage mode Admin
7. Si invalide → retour mode Vendeur/Conseil + notification erreur
8. Désactivation switch → retour mode Vendeur/Conseil + destruction token

Sécurité :

  • Mot de passe hashé avec bcrypt (backend)
  • Token de session cryptographiquement sécurisé
  • Pas de persistance du mode admin dans localStorage
  • Nettoyage automatique du token lors de la déconnexion
  • Modal de mot de passe avec support Escape + Enter

Événements émis :

  • 'auth:login' - Connexion admin réussie Payload: { mode: 'admin' }

  • 'auth:logout' - Déconnexion admin (retour vendeur) Payload: { mode: 'vendeur' }

  • 'auth:interface-updated' - Interface reconfigurée selon le mode Payload: { isGuestMode: boolean }

Événements écoutés :

  • 'app:initialized' - Charge le dashboard vendeur après init complète

Canaux IPC utilisés :

  • 'verify-password' - Vérification du mot de passe admin Paramètres: password (string) Retour: { success: boolean, sessionToken: string }
